# Threat Rakshaka - Android Malware Detection and Monitoring System

## Overview

Threat Rakshaka is an Android-based cybersecurity application designed to protect users from potentially malicious applications before installation. The system performs permission-based risk analysis on APK files, calculates risk scores, classifies applications based on threat severity, and prevents the installation of dangerous applications.

To support centralized monitoring and threat investigation, Threat Rakshaka integrates with a Police Monitoring Dashboard that receives and displays detected threat reports in real time.

---

## Problem Statement

Many Android users install APK files from untrusted sources without understanding the security risks associated with excessive or dangerous permissions. Malicious applications often exploit permissions such as Camera, Microphone, SMS, Accessibility Services, and Overlay Permissions to steal data, spy on users, or gain unauthorized control over devices.

Threat Rakshaka addresses this issue by analyzing applications before installation and proactively blocking potentially harmful software.

---

## Features

### APK Security Analysis

* Scans APK files before installation.
* Extracts permissions from the Android Manifest.
* Performs real-time security assessment.
* Generates a risk score based on predefined security rules.

### Permission-Based Risk Detection

Analyzes sensitive permissions including:

* Camera Access
* Microphone Access
* SMS Permissions
* Contacts Access
* Accessibility Services
* Overlay Permissions
* Storage Access
* Internet Access
* Install Packages Permission

### Suspicious Permission Combination Detection

Detects potentially dangerous combinations such as:

* Camera + Microphone
* SMS + Internet
* Accessibility Service Permissions
* Overlay + Internet Access
* Contacts + Internet Access
* Install Packages Permission

### Risk Classification

Applications are categorized into:

* Safe
* Suspicious
* Dangerous

### Installation Protection

* Allows installation of safe applications.
* Displays warnings for suspicious applications.
* Automatically blocks dangerous applications.

### Threat Reporting

* Sends threat reports to Supabase.
* Stores detected malicious application data.
* Supports centralized monitoring and investigation.

* Application Name
* Risk Level

Provides insights such as:

## System Workflow

1. User attempts to install an APK file.
2. Threat Rakshaka scans the APK before installation.
3. Permissions are extracted from the Android Manifest.
4. Risk score is calculated using predefined security rules.
5. Application is classified as Safe, Suspicious, or Dangerous.
6. Dangerous applications are blocked.
7. Threat information is uploaded to Supabase.
8. Police Dashboard displays reports for monitoring and investigation.
